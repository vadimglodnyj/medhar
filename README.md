# МедХар — медична характеристика (Excel only)

Веб-застосунок на Flask для генерації **медичної характеристики** з Excel-даних. Без SQLite та інших БД.

## Принцип роботи

- дані лише з файлів `.xlsx` у папці `data/`;
- усі `treatments_YYYY.xlsx` знаходяться автоматично (будь-який рік у назві, змінювати код щороку не потрібно);
- шаблон документа: `templates/medical_characteristic_template.docx`;
- після старту кеш даних прогрівається у фоні; на сторінці форми є індикатор готовності пошуку.

## Файли в `data/`

| Файл | Призначення |
|------|-------------|
| `treatments_YYYY.xlsx` | Річні вибірки. Назва **строго**: `treatments_` + 4 цифри року + `.xlsx` (наприклад `treatments_2026.xlsx`). Можна мати кілька років одночасно. |
| `treatments_final.xlsx` | Опційний архів. Якщо є, зчитується **першим**. |

**Злиття:** спочатку `treatments_final.xlsx` (якщо є), далі всі `treatments_YYYY.xlsx` у порядку **зростання року**. При дублікатах одного пацієнта пріоритет у даних із **більшого року** (останній файл у цьому порядку).

**Мінімум:** потрібен хоча б один джерело — або один і більше `treatments_YYYY.xlsx`, або `treatments_final.xlsx`.

Файли `*.xlsx` у `data/` за замовчуванням **не потрапляють у git** (див. `.gitignore`).

## Оновлення бази через інтерфейс

На сторінці медичної характеристики є блок **«Оновити базу за рік»**:

1. оберіть **рік** (за замовчуванням — поточний календарний);
2. завантажте `.xlsx`;
3. після перевірки структури старий `treatments_<рік>.xlsx` **замінюється** новим (атомарний запис), кеш скидається.

Обмеження розміру завантаження задається в `config.py` (`TREATMENTS_UPLOAD_MAX_BYTES`, за замовчуванням 80 МБ).

## Вимоги до структури Excel

Потрібні колонки **або** трійка **Прізвище**, **Ім'я**, **По батькові**, **або** колонка **ПІБ**. Файл має містити хоча б один рядок даних.

## Встановлення та запуск

```bash
pip install -r requirements.txt
python app.py
```

Або скрипт установки залежностей: `python install.py` (Windows: `install.bat`).

Відкрити в браузері: [http://127.0.0.1:5000/medical-characteristic](http://127.0.0.1:5000/medical-characteristic)

## API (для форми та інтеграцій)

| Метод | Шлях | Опис |
|--------|------|------|
| `GET` | `/` | Редірект на `/medical-characteristic` |
| `GET`, `POST` | `/medical-characteristic` | Форма та генерація DOCX |
| `GET` | `/api/treatments_ready` | Статус прогріву кешу (`ready`, `loading`, `error`) |
| `GET` | `/api/treatments_sources` | Список знайдених `treatments_YYYY.xlsx`, наявність `treatments_final` |
| `POST` | `/api/treatments_upload` | Форма: `file` (.xlsx), `year` (опційно) — заміна файлу року |
| `GET` | `/api/search_pib?q=...` | Підказки по ПІБ |
| `GET` | `/api/stats` | Кількість записів і унікальних пацієнтів |

## Безпека та продакшен

У `config.py` змініть `SECRET_KEY` на випадковий рядок і вимкніть `DEBUG` для публічного сервера. Завантаження Excel не захищені окремою авторизацією — розраховано на локальну / довірену мережу.

## Типові проблеми

- **Інший формат назви** — лише шаблон `treatments_YYYY.xlsx`, інакше файл не підхопиться автоматично.
